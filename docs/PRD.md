# CICPA 注册会计师每日学习助手 - 产品需求文档（PRD）

> 版本：v1.0
> 日期：2026-03-09
> 状态：初稿

---

## 一、产品概述

### 1.1 产品名称
**CPA Daily** —— 注册会计师每日学习助手

### 1.2 产品定位
面向中国注册会计师（CICPA）考试备考者的智能学习辅助系统。通过每日自动推送财经新闻关联考点讲解、真题解析和记忆辅助材料，帮助考生在碎片化时间中高效备考，实现"每日一学、日积月累"的学习目标。

### 1.3 目标用户
| 用户类型 | 特征描述 |
|---------|---------|
| 在职备考者 | 工作繁忙，学习时间碎片化，需要高效利用通勤、午休等间隙 |
| 全职备考者 | 需要系统化的复习节奏和知识点覆盖保障 |
| 二战/多战考生 | 已有基础但需要查漏补缺，避免重复学习已掌握内容 |

### 1.4 核心价值主张
1. **不用主动找题**：系统每天自动推送一个考点，结合当日财经新闻讲解，降低学习启动成本
2. **不会遗漏考点**：知识点去重和轮次机制，确保六科所有考点都被覆盖
3. **真题实战导向**：每个考点关联历年真题，直接对标考试出题方式
4. **记忆有方法**：AI自动生成助记口诀、对比表格、思维导图，科学记忆
5. **AI随时答疑**：任何不懂的考点可以随时向AI提问，获得专业解答

## 三、功能详细描述

### F01 每日学习提醒

**用户故事**：作为一名在职备考者，我希望每天早上收到微信提醒，这样我就不会忘记每日学习任务。

**功能描述**：
- 系统每天在用户设定的时间（默认07:30）通过微信推送学习提醒
- 推送内容包含：今日考点标题、所属科目、新闻标题摘要
- 用户点击推送消息可直接跳转到H5详情页查看完整内容
- 支持两种推送渠道：
  - **微信公众号模板消息**：一对一推送，需用户关注公众号
  - **企业微信机器人Webhook**：群推送，配置简单无需认证

**验收标准**：
- [ ] 每天准时推送，误差不超过1分钟
- [ ] 推送消息包含考点标题、科目、日期
- [ ] 点击跳转到H5详情页
- [ ] 推送失败自动重试3次
- [ ] 支持用户自定义推送时间

---

### F02 每日新闻采集

**用户故事**：作为备考者，我希望每天看到一条与注会考试相关的财经新闻，这样我可以在了解时事的同时联系考点。

**功能描述**：
- 系统每天06:00自动从财经新闻源抓取新闻
- 新闻来源：财联社、新浪财经、证券时报等主流财经媒体
- 自动去重和过滤，只保留与财务、审计、税务、经济法相关的新闻
- 每天选取一条最具考点关联价值的新闻用于当日推送
- 备选方案：如抓取失败，AI根据当前时事热点生成模拟财经新闻

**验收标准**：
- [ ] 每天至少成功抓取1条财经新闻
- [ ] 新闻内容与注会考试领域相关
- [ ] 新闻不重复（标题去重）
- [ ] 抓取失败有降级方案

---

### F03 考点讲解

**用户故事**：作为备考者，我希望系统能结合新闻热点来讲解考点，这样既有趣又能加深理解。

**功能描述**：
- AI分析当日新闻与哪个考点最相关，自动选定今日讲解考点
- 生成内容结构：
  1. **新闻解读**（2-3句话，说明新闻与考点的关联）
  2. **考点精讲**（800-1200字，系统深入讲解）
  3. **重点提醒**（3-5个考试易错要点）
- 内容语言严谨但通俗易懂，适合碎片化阅读
- 支持Markdown格式渲染

**验收标准**：
- [ ] 讲解内容与新闻有逻辑关联
- [ ] 考点内容准确，符合注会考试大纲
- [ ] 字数在800-1200字范围
- [ ] 包含至少3个重点提醒

---

### F04 真题关联

**用户故事**：作为备考者，我希望每个考点都能看到对应的历年真题，这样我可以了解考试出题方式。

**功能描述**：
- 每个考点关联1-3道历年真题
- 真题包含：题目内容、选项（选择题）、标准答案、详细解析
- AI对真题进行补充讲解，说明解题思路和易错点
- 支持的题型：单选、多选、计算分析、综合题、简答题

**验收标准**：
- [ ] 每日推送至少关联1道真题
- [ ] 真题与当日考点强相关
- [ ] 答案解析清晰完整

---

### F05 记忆辅助

**用户故事**：作为备考者，我希望系统能帮我创造记忆点，让我更容易记住知识点。

**功能描述**：
- AI针对每个考点自动生成三种记忆辅助材料：
  1. **助记口诀**：押韵、朗朗上口，方便背诵
  2. **对比表格**：将容易混淆的概念用表格对比（如权益法vs成本法）
  3. **思维导图**：用缩进列表表示知识结构，前端可渲染为导图
- 材料以Markdown格式输出，支持在页面中直接渲染

**验收标准**：
- [ ] 口诀押韵且内容准确
- [ ] 对比表格涵盖关键差异点
- [ ] 思维导图层次清晰，不超过3层

---

### F06 知识点去重

**用户故事**：作为备考者，我希望每天讲解的知识点不重复，确保我能在一个周期内覆盖所有考点。

**功能描述**：
- **轮次机制**：系统维护"覆盖轮次"，所有考点讲完一遍为一轮
- **去重追踪**：每次讲解后记录该考点已覆盖，当前轮次内不再重复
- **优先级排序**：未覆盖考点按 重要程度×0.6 + 考试频率×0.4 加权排序
- **科目均衡**：支持两种策略
  - 按周轮换：周一会计、周二审计、周三财管、周四经济法、周五税法、周六战略、周日随机
  - 随机均衡：随机选科目但保证各科均匀覆盖
- 全部覆盖后自动进入下一轮，第二轮可以从不同角度重新讲解

**验收标准**：
- [ ] 同一轮次内考点不重复
- [ ] 六科覆盖比例均衡
- [ ] 一轮结束后自动开始新轮次
- [ ] 优先讲解高频高重要考点

---

### F07 AI智能问答

**用户故事**：作为备考者，我在学习过程中遇到不懂的地方，希望能随时向AI提问并获得专业解答。

**功能描述**：
- 用户可以创建对话会话，针对某个考点或某个科目进行提问
- AI以注会辅导老师角色回答问题
- 支持多轮对话，保持上下文连贯
- 支持SSE流式输出，逐字显示回答
- 支持切换AI模型（Claude / OpenAI / 本地模型）
- 对话历史可保存和检索

**验收标准**：
- [ ] 回答内容专业、准确
- [ ] 流式输出延迟 < 2秒
- [ ] 对话上下文连贯
- [ ] 支持至少2种AI模型切换

---

### F08 个人笔记

**用户故事**：作为备考者，我希望在学习过程中随时记录笔记和心得，并能按科目分类管理。

**功能描述**：
- 用户可以在查看每日内容时快速添加笔记
- 笔记支持Markdown格式编辑
- 可关联到具体的考点或推送内容
- 支持标星/标记重点
- 支持按科目、标签、日期筛选
- 支持全文搜索

**验收标准**：
- [ ] 笔记创建/编辑/删除功能正常
- [ ] Markdown渲染正确
- [ ] 标星和筛选功能正常
- [ ] 搜索响应 < 1秒

---

### F09 文件存档

**用户故事**：作为备考者，我希望所有学习内容都自动保存为文件，方便我离线查看和整理。

**功能描述**：
- 每天23:00自动将当日推送内容存档为Markdown文件
- 文件目录结构：`/archive/{科目名}/{年份}/{月日}_{考点标题}.md`
- 每个文件包含完整的YAML元数据头（日期、科目、章节、考点、重要程度）
- 文件内容包含：新闻解读 + 考点精讲 + 关联真题 + 记忆辅助
- 维护全局索引文件 `index.json`，方便检索
- 支持手动下载单个文件或按科目批量导出

**存档文件模板**：
```markdown
---
date: 2026-03-09
subject: 会计
chapter: 第五章 长期股权投资
point: 权益法核算
importance: 5
---

# CPA每日学习 - 2026年3月9日

## 今日新闻
（新闻摘要）

## 考点精讲：权益法核算
（详细讲解内容）

## 关联真题
（真题及解析）

## 记忆辅助
（口诀、表格、导图）
```

**验收标准**：
- [ ] 文件按正确目录结构存储
- [ ] Markdown文件内容完整
- [ ] YAML元数据正确
- [ ] 索引文件实时更新
- [ ] 支持文件下载

### F10 考点浏览

- 显示各科覆盖进度百分比

**验收标准**：
- [ ] 树形结构正确展示
- [ ] 覆盖状态标记准确
- [ ] 进度百分比计算正确

## 五、非功能需求

### 5.1 性能需求
| 指标 | 要求 |
|------|------|
| 页面首屏加载 | < 2秒 |
| API响应时间 | < 500ms（非AI接口） |
| AI流式首token延迟 | < 2秒 |
| 定时任务执行 | 推送误差 < 1分钟 |
| 并发支持 | 单实例支持100并发用户 |

### 5.2 可用性需求
- 系统可用性 ≥ 99.5%
- AI服务不可用时有降级方案（切换备用模型或使用缓存内容）
- 新闻抓取失败时有AI生成备选方案
- 推送失败自动重试机制

### 5.3 安全需求
- 用户认证基于JWT Token
- API接口权限控制（管理员 vs 普通用户）
- AI API Key加密存储
- 微信推送配置脱敏显示
- 防止SQL注入和XSS攻击

### 5.4 数据需求
- 考点数据至少覆盖六科各30个核心考点（共≥180个）
- 真题数据至少涵盖最近5年（2021-2025）
- 每日新闻保留90天历史
- 存档文件永久保存
- 数据库每日自动备份

### 5.5 兼容性需求
- 用户端H5页面兼容：微信内置浏览器、Chrome、Safari
- 管理后台兼容：Chrome、Edge、Firefox最新版本
- 移动端适配：375px - 414px宽度屏幕

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI生成内容不准确 | 误导考生 | 提示词反复调优；标注"AI生成仅供参考"；支持用户反馈纠错 |
| 新闻API不稳定 | 无法获取新闻 | 多源备选；AI生成模拟新闻作为降级 |
| AI API费用过高 | 运营成本 | 支持本地模型Ollama；缓存已生成内容；控制token用量 |
| 微信推送限制 | 推送失败 | 多渠道备选；失败重试；提供Web端主动查看 |
| 考点数据录入工作量大 | 延迟上线 | 分科目逐步录入；支持Excel批量导入；AI辅助生成考点 |
